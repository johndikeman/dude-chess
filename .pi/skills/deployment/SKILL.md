# Deployment and Environment Skill

This skill contains information about the deployment method and environment of the `dude-chess` AI agent. This is useful for troubleshooting and understanding how the agent is managed.

## Overview

The `dude-chess` agent is deployed using **Nix** and **Home Manager**. It runs as a **systemd user service** on a VPS.

## Home Manager Setup

The deployment is managed via a `dotfiles` repository located at `/home/ubuntu/dude-workspace/dotfiles`.

- **`flake.nix`**: Defines the inputs (including `dude-chess` repo) and the `homeManagerConfiguration` for the `ubuntu` user.
- **`home.nix`**: Enables the `dude-chess` services:
  ```nix
  services.dude-chess.enable = true;
  services.dude-chess-checker = {
    enable = true;
    lichessUsername = "woaster";
    interval = "hourly";
  };
  ```

## Systemd Services

The services are defined in the `dude-chess` repository's `flake.nix` as a Home Manager module.

### `dude-chess.service`

The main bot service.

- **Type**: `simple`
- **User Service**: Managed by `systemctl --user`
- **Definition Location**: `/home/ubuntu/.config/systemd/user/dude-chess.service` (generated by Nix)
- **Working Directory**: `/home/ubuntu/dude-workspace/dude-chess`
- **Environment Variables**:
    - `DUDE_CONFIG_DIR`: `/home/ubuntu/.config/dude-chess`
    - `PI_SKILLS`: Points to the Nix store path containing bundled skills.
    - `WEB_BROWSE_BROWSER_BIN`: Path to Chromium for `pi-web-browse`.
    - `PATH`: Includes essential tools like `git`, `gh`, `nodejs`, `op`, and `chromium`.
- **Secrets**: Uses `op run` (1Password CLI) with a `.opvars` file to inject environment variables.

### `dude-chess-checker.service` & `timer`

A periodic task that checks for new Lichess games.

- **Type**: `oneshot`
- **Timer**: Runs according to the interval specified in `home.nix` (e.g., `hourly`).
- **Script**: `src/daily-check.js`
- **Secrets**: Also uses `op run`.

## Secrets Management

Secrets are not stored in the repository. They are managed using **1Password CLI (`op`)**.
- The `packages.default` in `flake.nix` copies a `.opvars` file into the package output.
- `op run --env-file <path-to-.opvars> -- <command>` is used to execute the agent with secrets.
- `GEMINI_JSON_TOKEN` is extracted and written to `~/.pi/agent/auth.json` during `ExecStartPre`.

## Troubleshooting

### Checking Status and Logs

```bash
# Check service status
systemctl --user status dude-chess.service
systemctl --user status dude-chess-checker.timer
systemctl --user status dude-chess-checker.service

# View logs
journalctl --user -u dude-chess.service -f
journalctl --user -u dude-chess-checker.service -f
```

### Restarting the Agent

```bash
systemctl --user restart dude-chess.service
```

### Applying Configuration Changes

If you change `home.nix` or the `flake.nix` in `dotfiles`:
```bash
cd ~/dude-workspace/dotfiles
home-manager switch --flake .
```

If you change the `dude-chess` code and want to redeploy:
1. Push changes to GitHub.
2. Update the `dude-chess.url` in `dotfiles/flake.nix` if necessary (e.g., update the ref).
3. Run `nix flake update dude-chess` in the `dotfiles` directory.
4. Run `home-manager switch --flake .`.

*Note: The agent can often self-update by being restarted if the latest code is already in the Nix store or if it pulls changes itself, but the official way is through Home Manager.*
